# Jerry on RTX 4090

Best use: the beast node. 24 GB VRAM handles 70B models, fast image generation
(SDXL, Flux), LoRA fine-tuning, and multi-modal tasks. If you have one of these,
make it a Jerry.

---

## Hardware

| Item | Spec |
|------|------|
| GPU | NVIDIA RTX 4090 (Ada Lovelace, 24 GB GDDR6X) |
| CUDA | 12.x (check `nvidia-smi`) |
| RAM | 32 GB+ system RAM recommended |
| OS | Windows 11, Ubuntu 22.04+, or Pop!_OS |
| Driver | 545+ for CUDA 12.3+ |

---

## What it can run

| Task | Feasible? | Notes |
|------|-----------|-------|
| 70B chat (llama3:70b-q4) | ✅ | ~22 GB VRAM, ~15–25 tok/s |
| 34B code (qwen2.5-coder:32b) | ✅ | ~20 GB VRAM, very fast |
| Image gen — SDXL | ✅ | ~6 GB VRAM, ~3s/image |
| Image gen — Flux.1 Dev | ✅ | ~16–20 GB VRAM, ~8s/image |
| Image gen — Flux.1 Schnell | ✅ | ~12 GB VRAM, ~2s/image |
| LoRA fine-tuning (7B) | ✅ | Use Kohya or Axolotl |
| Video gen (AnimateDiff) | ✅ | ~12 GB VRAM |
| Whisper transcription | ✅ | large-v3 in real-time |
| Multi-modal (llava:34b) | ✅ | ~22 GB VRAM |

## Installation (Linux — Ubuntu/Pop!_OS)

### 1 — NVIDIA drivers + CUDA

```bash
# Check existing driver
nvidia-smi

# If needed — install recommended driver
sudo ubuntu-drivers autoinstall
sudo reboot

# Verify CUDA toolkit
nvcc --version  # or install: sudo apt install nvidia-cuda-toolkit
```

### 2 — Node 22

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 3 — Ollama (CUDA auto-detected)

```bash
curl -fsSL https://ollama.com/install.sh | sh
# Verify GPU detected
ollama run llama3.2 --verbose 2>&1 | grep "using gpu"

# Pull heavy hitters
ollama pull llama3:70b-instruct-q4_0     # 39 GB download, fits in 24 GB VRAM
ollama pull qwen2.5-coder:32b            # code tasks
ollama pull nomic-embed-text             # embeddings
ollama pull llava:13b                    # vision tasks
```

### 4 — OpenClaw + his-and-hers

```bash
sudo npm install -g openclaw his-and-hers
tj --version
```

### 5 — Tailscale

```bash
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --authkey tskey-auth-YOUR_KEY
```

### 6 — tj onboard (Jerry role)

```bash
tj onboard
# Role: Jerry
# Name: jerry-beast (or your choice)
# Model: Ollama → llama3:70b-instruct-q4_0
# Tom's Tailscale IP: <from Tom's machine>
```

### 7 — Advertise capabilities

```bash
tj capabilities advertise
# Should detect: RTX 4090, 24 GB VRAM, CUDA 12.x
# Models: llama3:70b, qwen2.5-coder:32b, ...
```

## Installation (Windows 11)

### 1 — Prerequisites

- [NVIDIA Driver 545+](https://www.nvidia.com/Download/index.aspx)
- [Node 22 LTS](https://nodejs.org)
- [Tailscale](https://tailscale.com/download/windows)

### 2 — Ollama for Windows

Download and run the [Ollama installer](https://ollama.com/download/windows).
Ollama auto-detects CUDA on Windows — no extra setup needed.

```powershell
# Verify GPU is used
ollama run llama3.2 "hello"
# Check Ollama logs in %LOCALAPPDATA%\Ollama\logs\server.log
```

### 3 — OpenClaw + tj

```powershell
npm install -g openclaw his-and-hers
tj onboard
```

### 4 — Windows Firewall (allow gateway port)

```powershell
# Run as Administrator
$port = 3737  # default gateway port
New-NetFirewallRule -DisplayName "TJ Gateway" `
  -Direction Inbound -Protocol TCP -LocalPort $port -Action Allow
```

### 5 — Auto-start (Scheduled Task)

```powershell
# tj onboard handles this, but manual fallback:
$action = New-ScheduledTaskAction -Execute "cmd.exe" `
  -Argument "/c `"openclaw gateway start`""
$trigger = New-ScheduledTaskTrigger -AtLogOn
Register-ScheduledTask -TaskName "TJGateway" -Action $action -Trigger $trigger -RunLevel Highest
```

## Docker (CUDA)

For containerized deployments on Linux with NVIDIA Container Toolkit:

```bash
# Install nvidia-container-toolkit first:
# https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html

docker build \
  -t jerry-cuda:latest \
  -f docker/jerry/Dockerfile.cuda .

docker run -d --gpus all \
  -e TS_AUTHKEY=tskey-auth-... \
  -e JERRY_NAME="jerry-beast" \
  -e JERRY_EMOJI="🦾" \
  -e OLLAMA_MODELS="llama3:70b-instruct-q4_0,qwen2.5-coder:32b,nomic-embed-text" \
  jerry-cuda:latest
```

## Power notes

The 4090 draws up to 450W under load. For an always-on Jerry node, consider:

- **Schedule WOL** — keep it asleep, let Tom wake it only when needed (`tj wake`)
- **Power limit** — `sudo nvidia-smi -pl 350` (350W saves 20% power, ~3% perf loss)
- **Idle detection** — the entrypoint handles this; if no task in 30 min, gateway can signal Tom to allow sleep
