H1 can't reach the capabilities server or webhook endpoint on H2 if the Windows Firewall blocks the port. There was no wizard step for this.

**Fix:** Added `packages/cli/src/wizard/steps/firewall.ts`:
- Creates inbound TCP rule `HH-Gateway` for the configured gateway port
- Handles local Windows H2 (direct netsh + UAC fallback) and remote via SSH
- No-ops on Linux/macOS (Tailscale handles it)
- Prints manual `netsh` command on failure so setup can continue
- Wired in as Step 9 in `onboard.ts` (between autologin and startup)
