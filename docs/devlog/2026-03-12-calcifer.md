### v6 Loss Diagnostics (active run)

| Step | MSE | KL | Stats | Total |
|------|-----|----|-------|-------|
| 50 | 5.67 | 2.89 | 0.12 | ~6.0 |
| 70 | 5.76 | 1.23 | 0.12 | ~6.0 |
| 90 | 3.98 | 4.86 | 0.12 | 5.21 |
| 91 | ‚Äî | ‚Äî | ‚Äî | 6.51 |

**Observation:** KL converged well (2.89‚Üí1.23 steps 50‚Üí70) but MSE stayed flat. At step 90, MSE finally moved (5.76‚Üí3.98) but KL bounced back (1.23‚Üí4.86). The two objectives are fighting ‚Äî optimizer oscillating between local minima for each.

**Root cause analysis:**
1. KL uses `F.kl_div(reduction="batchmean")` ‚Äî correctly averaged. Not a bug.
2. MSE plateau is a landscape issue, not a weight calculation bug. The gradient through `_resample_tokens()` (linear interpolation from codec dim ‚Üí image token positions) may be smearing spatial geometry.
3. bf16 resolved the OOM issues from v4/v5 ‚Äî step timing stable at ~102s, no memory pressure.

## Inter-Agent Communication ‚Äî Permanent Channel

### Problem

GLaDOS's OpenClaw gateway was configured to bind port 18789, but Windows `svchost.exe` (IP Helper service) permanently holds that port. Gateway couldn't start.

### Solution

1. **GLaDOS config updated** ‚Äî port changed from 18789 ‚Üí 18790 in `~/.openclaw/openclaw.json`
2. **GLaDOS autostart** ‚Äî `OpenClaw-GLaDOS` scheduled task (runs at logon, 1 min delay)
3. **Calcifer persistent tunnel** ‚Äî `glados-tunnel.service` systemd user unit:
   - Forwards `127.0.0.1:19790` ‚Üí `100.119.44.38:18790`
   - Auto-restarts on failure (10s backoff)
   - Enabled on boot

### Sending Messages (Calcifer ‚Üí GLaDOS)

```bash
OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1 node ~/.openclaw/workspace/send-to-agent.js \
  ws://127.0.0.1:19790 \
  <glados_gateway_token> \
```

## Lost at Sea Baseline (TextMAS)

Results from Cofounder research baseline experiments. See `/home/ubuntu/hlca/baseline_notes.md` on Calcifer for full methodology.

| Mode | Team (L1‚Üì) | Best Individual | Delta |
|------|-----------|----------------|-------|
| Nominal (run 1) | 36 | 38 | +5.3% team wins |
| Nominal (run 2) | 38 | 36 | ‚àí5.3% best ind. wins |
| Interactive | 34 | 34 | 0% (tie, Agent 1 drag) |

**Finding:** Same model, different seeds (42/43). Interactive mode introduced conformity pressure ‚Äî Agent 0 (seed 42, scored 34 individually) was anchored to Agent 1's suboptimal output. Nominal mode aggregates, averaging noise. Interactive mode propagates it.

**Hypothesis for Vision Wormhole:** One-way bounded latent channel suppresses conformity pressure by design. If codec run shows team > best individual: three-point argument for structured latent comms as architectural principle.
