## Bug Fixed: TS2459 TypeCheck Failure

### Root cause
`packages/core/src/schedule/store.ts` imported `HHSchedule` as a value:
```typescript
import { HHSchedule, HHScheduleList } from "./schema.ts";
export type { HHSchedule };
```

`schema.ts` exports `HHSchedule` as both a Zod const AND a TypeScript type (same
identifier). TypeScript interpreted the local binding as the *value* (the Zod
schema), so `export type { HHSchedule }` was trying to re-export a locally-bound
value as a type — triggering TS2459 in `tsc --noEmit`.

### Fix
Split the import to use `import type` for the type-only binding:
```typescript
import { HHScheduleList } from "./schema.ts";
import type { HHSchedule } from "./schema.ts";
export type { HHSchedule };
```
