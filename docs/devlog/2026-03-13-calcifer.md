## Bug Fixed: `watch.ts` — spawn error handling

**Problem:** `runExecutor()` used a single-arg `new Promise((resolve) => ...)`. When a spawn fails (ENOENT — binary doesn't exist), the `child.on('error')` handler was calling `resolve()` with `{ success: false }`. This meant `poll()` treated it the same as a task that ran-and-failed, marking it `"failed"` permanently. The existing `catch` block that reverts to `"pending"` was never reached.

**Secondary bug:** `false` (and other fast-exiting executors) close their stdin before we finish writing task JSON → EPIPE on `child.stdin.write()`.

**Fix:**
- Changed to `new Promise((resolve, reject) => ...)` — spawn errors now call `reject()`
- `poll()` catch block catches the rejection → reverts task to `"pending"` for retry
- Added `child.stdin.on('error', () => {})` noop to suppress EPIPE noise
